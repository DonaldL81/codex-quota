# 项目逻辑说明

本文维护项目如何工作，包括运行链路、业务流程、数据流、接口和模块职责。它不维护版本流水账、开发规范或完整文件清单。

## 1. 总体运行链路

1. Tauri 启动主窗口，默认窗口不可见。
2. 初始化窗口状态，读取 `window-state.json` 中的模式、置顶、位置和大窗尺寸。
3. 初始化托盘图标和右键菜单。
4. 显示小窗并发出额度刷新请求。
5. 前端先尝试读取后端缓存 `last-quota.json`。
6. 前端调用 Rust 命令实时读取 Codex 额度。
7. Rust 启动 `codex.exe app-server --listen stdio://`。
8. Rust 通过 JSON-RPC 调用 `account/rateLimits/read`。
9. Rust 将结果归一化为前端需要的额度快照。
10. Rust 写入最后一次成功快照，前端更新 UI 和托盘图标。
11. 用户可通过托盘菜单切换小窗、大窗、置顶、自启动、自动刷新间隔、重启和退出。

## 2. 启动流程

启动时主要动作：

1. `src-tauri/src/lib.rs` 注册 Tauri 插件、窗口事件和命令。
2. `window::init_state` 从 app data 目录读取窗口状态。
3. `tray::init_tray` 创建托盘菜单和托盘图标。
4. 主窗口初始设置为小窗并触发 `quota-refresh-requested`。
5. 前端组件挂载后读取窗口状态、自启动状态、后端缓存并开始实时刷新。

启动失败通常表现为：

1. WebView2 缺失，窗口打不开或一闪而过。
2. 找不到 `codex.exe`，额度读取失败。
3. Codex 登录态过期或网络不可用，实时读取失败时显示“暂时无法获取”和具体原因。

## 3. 配置读取逻辑

| 配置来源 | 优先级 | 说明 |
|---|---|---|
| 环境变量 `CODEX_QUOTA_CODEX_PATH` | 最高 | 指定 `codex.exe` 完整路径 |
| `%LOCALAPPDATA%\OpenAI\Codex\bin\codex.exe` | 高 | Codex CLI 默认直接路径 |
| `%LOCALAPPDATA%\OpenAI\Codex\bin\<目录>\codex.exe` | 高 | 兼容 hash 或版本子目录 |
| PATH 中的 `codex.exe` | 兜底 | 直接运行 `codex.exe` |
| Tauri app data | 本地状态 | 保存窗口状态和最后额度快照 |

本项目不保存 Codex 密钥或账号密码。

## 4. 认证和权限逻辑

项目本身没有账号系统和权限系统。

额度读取依赖 Codex 本机登录态：

1. 用户需先安装并登录 Codex。
2. 工具启动本机 Codex app-server。
3. Codex app-server 使用本机已有登录状态访问额度信息。
4. 登录态过期、网络失败或 Codex 返回错误时，工具显示失败或继续保留最后缓存。

## 5. 核心业务流程

### 5.1 实时读取额度

1. 前端触发 `read_quota` 命令。
2. Rust 调用 `find_codex_path` 查找 `codex.exe`。
3. Rust 启动 `codex.exe app-server --listen stdio://`。
4. Rust 发送 `initialize` JSON-RPC。
5. Rust 发送 `account/rateLimits/read` JSON-RPC。
6. Rust 解析 `rateLimitsByLimitId.codex` 或 `rateLimits`。
7. Rust 计算 5 小时额度和周额度的剩余百分比。
8. Rust 写入 `last-quota.json`。
9. 前端更新小窗、大窗、托盘 tooltip 和托盘图标。

失败处理：

1. 找不到 Codex CLI 时返回明确错误。
2. app-server 启动失败、超时或返回错误时，前端显示“暂时无法获取”和具体原因。
3. 刷新失败时保留最后一次可用额度，并在状态区显示失败原因；没有可用额度时显示“暂时无法获取”。

### 5.2 读取缓存额度

1. 前端启动或切换模式时调用 `read_cached_quota`。
2. Rust 从 app data 目录读取 `last-quota.json`。
3. 前端兼容旧缓存字段并判断是否为可用额度。
4. 前端优先使用后端缓存和实时读取结果；WebView localStorage 只在后端缓存不存在时作为兜底。
5. 小窗和大窗都只从同一个展示快照渲染；只要已有有效额度，就显示百分比，不被 loading 状态遮盖。
6. 实时刷新成功后覆盖后端缓存、前端缓存和展示快照，并用前端当前时间更新“上次刷新”显示。

### 5.3 窗口和托盘交互

1. 左键点击托盘图标：显示、隐藏或置前窗口。
2. 右键菜单：打开小窗、打开大窗、置顶/取消置顶、开机自启动、自动刷新秒数、重启、版本号展示、退出。
3. 窗口显示时，短按单击或双击弹窗主体会调用 `hide_panel` 隐藏窗口；按住拖动超过阈值时执行窗口拖动，不隐藏窗口。
4. 大窗尺寸可调整，宽高被限制在项目定义范围内。
5. 小窗尺寸固定，适合长期常驻。
6. 窗口位置、大窗尺寸、置顶状态和模式会写入 `window-state.json`。

## 6. 数据流说明

| 数据 | 来源 | 处理位置 | 保存位置 | 输出 |
|---|---|---|---|---|
| 额度原始数据 | Codex app-server | `quota.rs` | 不直接保存 | 归一化额度快照 |
| 额度快照 | `quota.rs` | Rust 和前端 | `last-quota.json`、localStorage、前端最后可用快照 | 小窗、大窗、托盘 |
| 窗口状态 | 用户交互和 Tauri 窗口事件 | `window.rs` | `window-state.json` | 下一次启动恢复 |
| 自启动状态 | Tauri autostart 插件 | 前端和托盘菜单 | 系统自启动配置 | 菜单勾选状态 |
| 自动刷新秒数 | 右键菜单预设项 | 前端和托盘菜单 | localStorage | 自动刷新定时器和菜单勾选状态 |

## 7. 接口或命令概览

| 入口 | 类型 | 作用 | 是否需要认证 |
|---|---|---|---|
| `read_quota` | Tauri command | 实时读取 Codex 额度 | 依赖 Codex 本机登录态 |
| `read_cached_quota` | Tauri command | 读取最后一次成功额度 | 否 |
| `show_panel` / `hide_panel` | Tauri command | 显示或隐藏窗口 | 否 |
| `set_mode` | Tauri command | 切换小窗/大窗 | 否 |
| `toggle_topmost` | Tauri command | 切换置顶状态 | 否 |
| `remember_window_state` | Tauri command | 保存窗口状态 | 否 |
| `update_tray_quota` | Tauri command | 更新托盘图标和 tooltip | 否 |
| `app.restart()` | Tauri API | 托盘菜单触发应用重启 | 否 |
| `codex.exe app-server --listen stdio://` | 外部进程 | 提供 Codex JSON-RPC app-server | 依赖 Codex 登录态 |

## 8. 模块职责

| 模块或文件 | 职责 | 备注 |
|---|---|---|
| `src/App.svelte` | UI 状态、刷新、缓存读取、窗口交互和自启动开关 | 小窗和大窗共用同一状态 |
| `src/main.ts` | 前端入口 | 挂载 Svelte 应用 |
| `src/styles/app.css` | 视觉样式 | 控制小窗、大窗和进度条展示 |
| `src-tauri/src/lib.rs` | Tauri 命令注册和应用启动 | 连接前端、窗口、托盘和额度模块 |
| `src-tauri/src/quota.rs` | Codex 路径查找、app-server 调用、额度解析和缓存 | 额度读取核心 |
| `src-tauri/src/window.rs` | 窗口模式、位置、尺寸、置顶和持久化 | 包含旧尺寸迁移 |
| `src-tauri/src/tray.rs` | 托盘菜单、托盘点击、图标绘制和 tooltip | 图标用两条状态条展示额度 |
| `scripts/release-version.ps1` | 同步版本号 | 必须保持 UTF-8 无 BOM 写出 |
| `scripts/verify-release.ps1` | 发布包校验 | 包含便携版启动、WebView2、Codex 读取等检查 |

## 9. 日志和历史逻辑

项目默认不维护长期业务日志。

持久化文件：

1. `window-state.json`：窗口位置、模式、大窗尺寸和置顶状态。
2. `last-quota.json`：最后一次成功额度快照。
3. `localStorage`：前端兜底缓存。

这些数据位于 Tauri app data 目录，不应提交到仓库。

## 10. 错误处理逻辑

1. 找不到 Codex CLI：提示安装 Codex Desktop 或设置 `CODEX_QUOTA_CODEX_PATH`。
2. app-server 启动失败：提示无法启动 Codex app-server。
3. app-server 输出读取失败：提示 stdout 或 JSON-RPC 读取失败。
4. app-server 超时：15 秒超时并返回错误。
5. Codex 返回错误：前端归类为网络异常、账号未登录/登录过期、超时、无额度数据或原始错误原因。
6. 缓存读取失败：静默忽略，不阻塞实时读取。
7. 缓存写入失败：静默忽略，不阻塞主流程。

## 11. 前端或交互逻辑

前端只有一个主页面，根据模式展示小窗或大窗：

1. 小窗显示 5 小时额度和周额度百分比。
2. 大窗显示更新时间、重置时间和两条进度条。
3. 额度颜色规则：低于或等于 20% 为红色，21%-50% 为橙色，51%-100% 为绿色。
4. 自动刷新间隔使用右键菜单预设值，支持 `30s`、`1min`、`5min`、`10min`、`20min`、`30min`、`60min`。
5. 手动刷新由右键菜单触发。
6. 切换窗口模式时优先复用前端最后可用快照，再补读后端缓存并发起实时刷新。
7. 短按单击或双击弹窗主体时隐藏窗口，按住拖动时不隐藏；按钮和右下角缩放手柄不触发隐藏动作。
