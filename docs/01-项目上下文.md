# 项目上下文

本文维护项目总览、背景、关键决策、当前状态和技术债。它用于帮助新会话快速理解项目，不记录详细代码流程、开发规范或一次性排查过程。

## 1. 项目目标

Codex 额度监控是一个 Windows 桌面托盘小工具，用来实时查看 Codex 账号的剩余额度。

项目重点：

1. 体积小，便携版约 5 MB。
2. UI 简单，只显示关键额度信息。
3. 系统占用低，适合常驻托盘或桌面角落。
4. 支持小窗常驻和大窗查看完整信息。
5. 不依赖打开 Codex 桌面窗口，可以直接调用本机 Codex app-server 读取额度。

不解决的问题：

1. 不管理 Codex 账号登录。
2. 不修改 Codex 额度或订阅状态。
3. 不替代 Codex 官方客户端。
4. 不提供跨平台版本，当前目标是 Windows。

## 2. 用户和使用场景

| 使用者 | 主要场景 | 关键需求 |
|---|---|---|
| Codex 日常用户 | 写代码时随时查看 5 小时额度和周额度 | 快速、低打扰、常驻可见 |
| 多设备用户 | 在不同 Windows 电脑上使用 Codex | 自动找到本机 Codex CLI，减少路径配置 |
| 发布测试者 | 验证不同机器上的窗口和额度显示 | 便携版可直接复制运行 |

## 3. 技术栈

| 层级 | 当前方案 | 说明 |
|---|---|---|
| 前端 | Svelte 5 + TypeScript + Vite | 渲染小窗、大窗、状态和托盘事件响应 |
| 桌面容器 | Tauri 2 | Windows 桌面窗口、托盘、开机自启动和打包 |
| 后端 | Rust + Tokio | 启动 Codex app-server，通过 stdio JSON-RPC 读取额度 |
| 数据存储 | Tauri app data 目录 + WebView localStorage | 保存窗口状态和最后一次成功额度快照 |
| 构建和发布 | npm scripts + Tauri CLI + PowerShell 脚本 | 生成便携版和 NSIS 安装包 |
| 验证 | `npm run typecheck`、`cargo check`、`scripts/verify-release.ps1` | 覆盖前端类型、Rust 编译和发布包冒烟 |

## 4. 架构概览

项目由 `codex-quota-v2/` 承载当前 V2 源码：

1. 前端 `src/App.svelte` 负责窗口 UI、状态渲染、自动刷新和用户交互。
2. Rust `src-tauri/src/quota.rs` 负责寻找 `codex.exe`、启动 `app-server`、读取额度、归一化字段并写入 `last-quota.json`。
3. Rust `src-tauri/src/window.rs` 负责小窗/大窗尺寸、位置、置顶、状态持久化和旧尺寸迁移。
4. Rust `src-tauri/src/tray.rs` 负责托盘菜单、托盘点击、托盘图标和 tooltip。
5. `scripts/*.ps1` 负责开发、构建、发布、版本同步和发布校验。

## 5. 运行和部署形态

| 形态 | 说明 | 入口 |
|---|---|---|
| 开发版 | 本地启动 Vite 和 Tauri dev | `npm run start` 或 `scripts/dev.ps1` |
| 便携版 | 单文件 EXE，适合直接复制测试或使用 | `dist-portable/Codex Quota Monitor <version> Portable.exe` |
| 安装版 | NSIS 安装包，适合固定安装 | `src-tauri/target/release/bundle/nsis/*.exe` |
| GitHub 发布 | 仓库存 V2 源码，Release 挂 EXE 附件 | `DonaldL81/codex-quota` |

## 6. 关键设计决策

| 决策 | 原因 | 影响 |
|---|---|---|
| 只公开发布 V2 源码 | V0/V1 是历史探索版本，公开仓库应保持清爽 | GitHub 发布范围以 `codex-quota-v2/` 为主 |
| 使用 Tauri 2 重构 | 需要轻量桌面常驻工具，避免 Electron 包体过大 | 便携版约 5 MB |
| 直接调用 Codex app-server | 关闭 Codex 桌面窗口后仍可读取额度 | 依赖本机 `codex.exe`、登录态和网络 |
| 后端保存最后一次成功额度 | 前端 localStorage 在重启、热更新、首次安装时不够稳定 | 小窗和大窗有共同兜底数据 |
| 兼容 Codex hash 子目录安装路径 | 不同电脑 Codex CLI 路径可能不同 | 自动搜索 `%LOCALAPPDATA%\OpenAI\Codex\bin\<目录>\codex.exe` |
| README 保持用户视角 | GitHub 首页面向下载和使用者 | 开发细节进入 `docs/` |
