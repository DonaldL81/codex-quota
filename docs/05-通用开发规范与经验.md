## 1. 项目内维护规则

1. 项目专属开发规则写入 `docs/03-项目开发规范.md`。
2. 项目踩坑和解决方案写入 `docs/04-项目经验积累.md`。
3. 可跨项目复用的经验追加到全局通用规范文档。
4. 如全局规范更新影响当前项目，应在相关项目文档中补充项目侧执行要求。

## 2. 当前项目适用的通用要求

1. 修改前先检查当前真实文件状态和 Git 状态。
2. 不覆盖用户已有未提交改动。
3. 不提交依赖目录、构建产物、日志、缓存和本地运行时数据。
4. 修改后必须运行与风险匹配的验证。
5. 无法验证时必须说明原因和剩余风险。
6. README 保持用户视角，开发细节进入 `docs/`。
