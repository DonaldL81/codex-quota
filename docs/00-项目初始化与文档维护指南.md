# 项目初始化与文档维护指南

本文是本项目新会话启动、文档职责划分和后续文档维护的总入口。它只维护流程和边界，不记录具体代码细节或一次性排查过程。

## 1. 文档结构

| 文件 | 作用 | 职责边界 |
|---|---|---|
| `README.md` | 面向用户的使用说明 | 只写用户如何理解、下载、运行和排查常见问题 |
| `docs/00-项目初始化与文档维护指南.md` | 新会话启动、文档职责和维护规则 | 不写业务逻辑、版本流水账或具体排查过程 |
| `docs/01-项目上下文.md` | 项目目标、技术栈、关键决策和版本状态 | 不写详细代码流程和开发规范 |
| `docs/02-项目逻辑说明.md` | 启动链路、数据流、模块职责和交互逻辑 | 不写版本流水账和完整文件清单 |
| `docs/03-项目开发规范.md` | 当前项目的开发、验证、发布和安全规则 | 不沉淀一次性排查过程 |
| `docs/04-项目经验积累.md` | 当前项目踩坑、难点、解决方案和排查路径 | 不重复通用开发规则 |
| `docs/05-通用开发规范与经验.md` | 引用跨项目通用规范并记录同步要求 | 不复制通用规范全文 |

## 2. 新会话启动流程

同一项目开启新会话时，按以下顺序读取：

1. `README.md`，理解用户视角的用途和使用方式。
2. `docs/00-项目初始化与文档维护指南.md`，确认文档边界。
3. `docs/01-项目上下文.md`，理解项目目标、当前版本和关键决策。
4. `docs/02-项目逻辑说明.md`，理解额度读取、缓存、窗口和托盘链路。
5. `docs/03-项目开发规范.md`，确认运行、验证、发布和提交边界。
6. `docs/04-项目经验积累.md`，查看已知问题和排查路径。
7. 按任务读取相关源码、脚本或配置。
8. 检查 `git status --short`，识别已有未提交改动。

## 3. 文档更新规则

1. 用户下载、运行或使用方式变化，更新 `README.md`。
2. 项目目标、技术栈、运行形态、关键决策或版本状态变化，更新 `docs/01-项目上下文.md`。
3. 启动流程、数据流、模块职责、接口命令或交互逻辑变化，更新 `docs/02-项目逻辑说明.md`。
4. 开发命令、验证命令、发布流程、忽略规则或安全边界变化，更新 `docs/03-项目开发规范.md`。
5. 当前项目踩坑、原因、解决方案或排查路径变化，更新 `docs/04-项目经验积累.md`。
6. 可跨项目复用的经验，追加到 `D:\Desktop\AI\_workspace\知识库\01_项目开发沉淀\跨项目通用开发规范与经验.md`，并在 `docs/05-通用开发规范与经验.md` 记录同步状态。

## 4. README 维护边界

README 必须保持用户视角，重点说明：

1. 项目是什么。
2. 适合谁使用。
3. 如何下载、启动和使用。
4. 常见问题如何处理。
5. 用户可感知的版本变化。

README 不写内部架构、模块拆分、测试记录、重构记录、技术债和开发过程。

## 5. 发布目录约定

GitHub 仓库公开内容以 V2 为主：

1. 根目录 `README.md`。
2. 根目录当前发布用 EXE。
3. `codex-quota-v2/` 的源码、配置、脚本和必要截图。

本地历史目录 `codex-quota-v0/`、`codex-quota-v1/` 和本地开发文档默认不作为公开发布范围。
