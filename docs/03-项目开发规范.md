# 项目开发规范

本文维护当前项目必须遵守的开发规则，包括运行、测试、发布、安全边界、依赖和文档同步。它不记录一次性开发过程，也不详细说明业务逻辑。

## 1. 基本原则

1. 普通用户视角内容写入根目录 `README.md`。
2. 项目开发上下文写入 `docs/01` 到 `docs/04`。
3. 跨项目通用经验以 `D:\Desktop\AI\跨项目通用开发规范与经验.md` 为准。
4. 修改功能代码前先检查 `git status --short`，不得覆盖用户已有改动。
5. 对用户明确说“不要动开发代码”的任务，只允许修改 README、docs 或其他明确文档文件。

## 2. 本项目运行方式

| 场景 | 命令或入口 | 说明 |
|---|---|---|
| 本地开发启动 | `npm run start` | 在 `codex-quota-v2/` 下运行 Tauri dev |
| 前端构建 | `npm run frontend:build` | 生成前端 `dist/` |
| 便携版构建 | `npm run build:portable` 或 `scripts/build-portable.ps1` | 生成无安装 EXE |
| 安装版构建 | `npm run build` 或 `scripts/build.ps1` | 生成 NSIS 安装包 |
| 发布校验 | `scripts/verify-release.ps1` | 校验版本、EXE、WebView2、Codex 路径和额度读取 |

命令默认工作目录为：

```text
D:\Desktop\AI\codex_quota\codex-quota-v2
```

## 3. 本项目测试和验证

修改后按影响范围运行验证：

| 修改范围 | 必跑验证 |
|---|---|
| 前端 UI 或状态 | `npm run typecheck` |
| Rust/Tauri 代码 | `cargo check`，工作目录 `codex-quota-v2/src-tauri` |
| 额度读取逻辑 | `cargo check` + 真实 Codex app-server 读取验证 |
| 窗口尺寸或状态 | `npm run typecheck` + `cargo check` + 开发版预览 |
| 构建或发布 | `npm run typecheck` + `cargo check` + `scripts/verify-release.ps1` |
| 版本号 | 检查 `package.json`、`package-lock.json`、`src-tauri/Cargo.toml`、`src-tauri/tauri.conf.json`、`quota.rs` 客户端版本 |
| 文档 | 使用 UTF-8 读取确认，检查是否仍符合文档职责边界 |

无法运行应有验证时，交付说明必须说明原因和剩余风险。

## 4. 本项目发布规则

发布前确认：

1. 版本号同步。
2. README 已同步用户可见变化。
3. `docs/01` 到 `docs/04` 已同步开发相关变化。
4. 根目录发布文件名和 README 下载说明一致。
5. `.gitignore` 放行规则与实际要提交的根目录 EXE 一致。
6. 不提交 `node_modules/`、`dist/`、`dist-portable/`、`src-tauri/target/`、日志和本地缓存。
7. `scripts/verify-release.ps1` 通过。
8. 发布到 GitHub Release 时，便携版和安装版附件名称清晰标注版本。

## 5. 本项目配置和运行时数据

| 类型 | 默认位置 | 是否进入版本控制 | 是否进入发布包 |
|---|---|---|---|
| Codex CLI 路径环境变量 | `CODEX_QUOTA_CODEX_PATH` | 否 | 否 |
| 窗口状态 | Tauri app data 下 `window-state.json` | 否 | 否 |
| 额度缓存 | Tauri app data 下 `last-quota.json` | 否 | 否 |
| 前端缓存 | WebView localStorage | 否 | 否 |
| 构建产物 | `dist/`、`dist-portable/`、`src-tauri/target/` | 否，除明确发布文件 | 视发布目标而定 |

本项目不保存 Codex 密钥、账号密码或用户 token。

## 6. 本项目安全边界

| 边界类型 | 当前项目规则 | 备注 |
|---|---|---|
| 允许处理的目录 | `D:\Desktop\AI\codex_quota` 及其项目文件 | 发布和文档任务默认只在此范围内操作 |
| 禁止处理的目录 | 用户未授权的其他项目目录、系统敏感目录 | 读取 Codex 安装路径只用于定位可执行文件 |
| 删除入口 | 无用户数据删除功能 | 清理构建产物或进程前必须确认目标 |
| 密钥保存位置 | 不保存密钥 | 依赖 Codex 本机登录态 |
| 日志脱敏要求 | 不输出账号 token、密钥或私有认证信息 | 错误信息可包含本机路径，但发布文档避免泄露私人路径 |

## 8. 本项目文档同步规则

| 场景 | 额外需要同步的文档 | 说明 |
|---|---|---|
| 用户可见功能变化 | `README.md`、`docs/01-项目上下文.md` | README 写用户收益，docs 写开发背景 |
| 额度读取流程变化 | `docs/02-项目逻辑说明.md`、`docs/04-项目经验积累.md` | 需要记录数据流和排查路径 |

## 9. 编码和文件编辑要求

1. PowerShell 脚本尽量使用 ASCII 提示文本，避免 Windows PowerShell 5 对无 BOM 中文脚本解析异常。
2. JSON、Rust、TypeScript 和 Markdown 文件使用 UTF-8。
3. Tauri 配置 JSON 不得带 UTF-8 BOM。
4. 版本脚本写文件时必须使用 UTF-8 无 BOM。
5. Rust 格式化应按项目 edition 执行，避免单文件默认 edition 导致解析异常。
6. 前端状态逻辑修改后，必须检查小窗和大窗是否复用同一份额度数据。

## 10. 应急处理

1. 旧会话卡在 `inProgress` 时，不等待旧进程恢复；读取历史会话作为开发记录，并以当前文件状态为准。
2. 另一台电脑读取失败时，优先排查 `codex.exe` 路径和 `app-server` 是否能启动。
3. 小窗显示“正在读取”但大窗有数据时，优先排查缓存复用和 loading 状态覆盖。
4. 打包失败且报 JSON 解析异常时，优先检查 `tauri.conf.json` 是否带 BOM。
